使用hexo和github搭建博客

人脑相当于cpu,没有内存,只有若干寄存器.所以需要些地方记录些什么

准备工作

  • github账号
  • node.js、npm
  • git for windows

github 账号配置

新建名为用户名.github.io的仓库

安装 hexo

npm install hexo -g

检查版本hexo -v

输入 hexo init 初始化该文件夹

输入 npm install 安装所需要的组件

输入 hexo g 体验hexo

输入 hexo start 开启服务器

将hexo与github page 联系起来

输入 cd ~/.ssh 检查是否有 .ssh文件夹

输入 ssh-keygen -t rsa -C "email@email.com"生成密钥，最后得两个文件 id_res和 id_rsa.pub

输入 eval "$(ssh-agent-s)"添加密钥到ssh-agent

输入 ssh-add ~./ssh/id_rsa 添加生成得SSH key到ssh-agent

登录github 点击图像下得setting 添加id_rsa.pub内容

输入ssh -T `git@github.com`测试ssh是否成功 邮箱地址不用改

配置 _config.yml

配置Deployment

新建一篇博客

hexo new pist "博客名" 会在_posts目录下创建文件

安装扩展 npm install hexo-deployer-git --save

发布hexo d -g

多台电脑更新github pages博客

思路: 使用git分支

新建一个分支,将hexo环境目录文件以及md源文件保存到新建的分支上. 具体操作如下:

  • 在username.githgub.io仓库上新建分支并拢到本地
    在GitHub上新建一个分支,并在该仓库->setting-Branches 里设置新分支为默认分支. 清空里面的文件. 将新分支clone到本地. 将原来本地hexo环境目录文件拷贝到新分支.然后提交 注意:删除themes文件夹里的.git(如果有)

  • 在新环境下clone新分支到本地,然后执行nmp install恢复环境

日常高频操作: hexo g hexo d
